#### `lib/core/` — Configuration & Constants

* **`config.dart`**: Evaluates the running environment using `kDebugMode` to alternate between local emulator loops (`10.0.2.2:8000`) and the hardwired local network production IPs.
* **`constants.dart`**: Houses `StorageKeys` to keep local read/write strings consistent across secure storage layers.
* **`app_exceptions.dart`**: Decodes unsuccessful status codes (400, 404, 409) and pulls the raw backend `detail` property out of the payload response to show friendly errors in the UI.

#### `lib/models/` — Serialized Data Entities

Data serialization handles back-and-forth conversions via `json_serializable` and `build_runner`. Individual schema properties use explicit key annotations to clean up incoming database fields:

| Dart Model | JSON Field Mappings | Purpose / Function |
| --- | --- | --- |
| `Asset` | `asset_id`, `aztec_code`, `asset_status` | Maps underlying hardware profiles to local objects. |
| `AssetListResponse` | `items`, `total` | Handles index list collections and total counts. |
| `CheckoutRequest` | `aztec_code` | Validation layer tracking barcode scan strings. |
| `ReturnInitiateRequest` | `aztec_code`, `kiosk_id` | Ties an incoming return asset to a specific station origin. |

#### `lib/services/api/` — Network Layer

The app handles active requests through `api_client.dart`, an interface generated by Retrofit. Outbound connections have a strict 10-second handshake timeout limit and a 30-second response wait threshold to keep the screen from hanging indefinitely during high network load.

| Method Name | Verb | Endpoint | Authentication Requirement |
| --- | --- | --- | --- |
| `nfcLogin()` | POST | `/api/v1/auth/nfc` | Open access, public endpoint to check user badges. |
| `pinLogin()` | POST | `/api/v1/auth/pin` | Verifies user PINs and returns access and refresh token objects. |
| `getCatalog()` | GET | `/api/v1/catalog` | Role-aware route returning categorized asset availability counts. |
| `checkout()` | POST | `/api/v1/loans/checkout` | Protected checkout action using explicit idempotency tokens. |
| `returnInitiate()` | POST | `/api/v1/loans/return/initiate` | Hands over target codes along with kiosk identifiers. |
| `getLoanStatus()` | GET | `/api/v1/loans/{id}/status` | Light polling checkpoint tracking back-end loan state changes. |
| `logout()` | POST | `/api/v1/auth/logout` | Revokes the active refresh token session on the backend database. |

## PART 2: Feature Specifications

### 2.1 Authentication & Token Lifecycle

#### The Login Flow

Authentication uses a two-phase check to manage terminal access safely. The user taps their badge, validation passes, the PIN pad opens, and then session keys are written to storage:

```dart
// 1. Validate badge identification sequence
await client.nfcLogin(NfcLoginRequest(nfcTagId: nfcTagId));

// 2. Authorize terminal profile via input PIN sequence
final tokenResponse = await client.pinLogin(
  PinLoginRequest(nfcTagId: nfcTagId, pin: pin),
);

// 3. Commit encryption variables safely to hardware storage
await storage.saveTokens(
  accessToken: tokenResponse.accessToken,
  refreshToken: tokenResponse.refreshToken,
);

// 4. Cache verified profile parameters locally
final user = await client.getMe();
await storage.saveUser(jsonEncode(user.toJson()));

```

#### Safe Logouts

Logouts clean up state on both sides of the application. The system attempts to safely notify the remote server to revoke the refresh token, then completely flushes the local encrypted storage partition within a guaranteed `finally` block:

```dart
Future<void> logout() async {
  try {
    final refreshToken = await storage.getRefreshToken();
    if (refreshToken != null) {
      await client.logout(RefreshTokenRequest(refreshToken: refreshToken));
    }
  } catch (_) {
    // Suppress network drops to ensure local device cleanup always executes
  } finally {
    await storage.clearAll();
  }
}

```

## PART 3: Security, Idempotency & Kiosk Hardening

### 3.1 Token Refresh Strategy — `AuthInterceptor`

Outbound API traffic passes through a unified `AuthInterceptor`. If a request fails with a `401 Unauthorized` code, the interceptor halts the traffic queue, initiates an independent token update call, rewrites local security storage keys, and transparently completes the original user transaction.

```dart
class AuthInterceptor extends Interceptor {
  final Ref _ref;
  AuthInterceptor(this._ref);

  @override
  void onRequest(RequestOptions options, RequestInterceptorHandler handler) async {
    final storage = _ref.read(secureStorageProvider);
    final token = await storage.getAccessToken();
    if (token != null) {
      options.headers['Authorization'] = 'Bearer $token';
    }
    handler.next(options);
  }

  @override
  void onError(DioException err, ErrorInterceptorHandler handler) async {
    // Never run intercepts against auth routes to block feedback loops
    if (err.requestOptions.path.contains('/auth/')) {
      return handler.next(err);
    }

    if (err.response?.statusCode == 401) {
      final refreshed = await _ref.read(authServiceProvider).refreshToken();
      if (refreshed) {
        final storage = _ref.read(secureStorageProvider);
        final token = await storage.getAccessToken();
        
        final opts = err.requestOptions;
        opts.headers['Authorization'] = 'Bearer $token';
        
        final dio = _ref.read(dioProvider);
        return handler.resolve(await dio.fetch(opts));
      }
    }
    handler.next(err);
  }
}

```

### 3.2 Double-Tap Protection (`Idempotency-Key`)

To keep an impatient user from multi-tapping buttons on the screen and triggering duplicate reservations or lock contention errors, mutate actions require a unique `Idempotency-Key` header:

```dart
final idempotencyKey = const Uuid().v4();
final response = await client.checkout(
  CheckoutRequest(aztecCode: aztecCode),
  idempotencyKey,
);

```

### 3.3 Kiosk Inactivity Deadman Switch — `_IdleTimerWrapper`

To prevent session hijacking if a user forgets to close their profile before walking away from the terminal, a translucent `Listener` wraps the root layout to monitor and reset a 60-second inactivity countdown timer.

```dart
class _IdleTimerWrapper extends ConsumerStatefulWidget {
  final Widget child;
  const _IdleTimerWrapper({required this.child});

  @override
  ConsumerState<_IdleTimerWrapper> createState() => _IdleTimerWrapperState();
}

class _IdleTimerWrapperState extends ConsumerState<_IdleTimerWrapper> {
  Timer? _idleTimer;
  static const _idleDuration = Duration(seconds: 60);

  @override
  void initState() {
    super.initState();
    _resetTimer();
  }

  void _resetTimer() {
    _idleTimer?.cancel();
    _idleTimer = Timer(_idleDuration, _handleIdle);
  }

  void _handleIdle() {
    ref.read(authProvider.notifier).logout();
    // Route via Riverpod router wrapper to bypass layout builder context limitations
    ref.read(routerProvider).go('/login');
  }

  @override
  void dispose() {
    _idleTimer?.cancel();
    super.dispose();
  }

  @override
  Widget build(BuildContext context) {
    return Listener(
      behavior: HitTestBehavior.translucent,
      onPointerDown: (_) => _resetTimer(), // Intercepts touches anywhere on screen
      child: widget.child,
    );
  }
}

```
